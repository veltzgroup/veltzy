## Achados por Dimensao

### 1. FUNCIONAL

**🔴 F-01: Email de convite nao e enviado**
- **Arquivo:** `src/components/sellers/invite-member-modal.tsx:40-44`
- **Problema:** Convite e criado no banco e link gerado, mas o link so aparece na UI. Nenhum email e disparado.
- **Impacto:** Admin precisa copiar link manualmente e enviar por WhatsApp/email. Inaceitavel para producao.
- **Acao:** Criar Edge Function `send-invite-email` com Brevo API + trigger no INSERT da tabela invitations.

**🟠 F-02: Sem prevencao de convites duplicados**
- **Arquivo:** `src/services/team.service.ts:26-35`
- **Problema:** Mesmo email pode ser convidado multiplas vezes, criando N convites pendentes.
- **Acao:** Verificar se ja existe convite pendente para o email antes de criar novo. Auto-revogar anterior se existir.

**🟠 F-03: Race condition na aceitacao**
- **Arquivo:** `src/pages/aceitar-convite.tsx:98-109`
- **Problema:** Double-click pode criar user_role duplicado. INSERT em user_roles nao tem check de conflito.
- **Acao:** Usar UPSERT ou RPC com transaction. Desabilitar botao durante processamento (ja tem, mas race possivel).

**🟡 F-04: Funcao acceptInvite no service nunca e usada**
- **Arquivo:** `src/services/team.service.ts:85-89`
- **Problema:** RPC `accept_invite` espera `invite_code` (string) mas tabela usa `token` (UUID). Pagina reimplementa logica inline.
- **Acao:** Deletar funcao morta ou migrar logica da pagina para o service.

### 2. DADOS

**🟠 D-01: Tabela abandonada company_invites coexiste com invitations**
- **Arquivo:** `supabase/migrations/006_team_reports.sql`, `011_central_complement.sql`
- **Problema:** Duas tabelas de convite no banco. RPC referencia a antiga. Codigo usa a nova.
- **Acao:** Migration para DROP TABLE company_invites se nenhum dado existe.

**🟡 D-02: Indices ausentes nas queries frequentes**
- **Problema:** Sem indices em `invitations(token)`, `invitations(company_id, status)`, `invitations(email, company_id)`.
- **Acao:** Criar indices na proxima migration.

**🟡 D-03: Sem constraint UNIQUE em (user_id, company_id) no user_roles**
- **Problema:** Race condition pode criar role duplicado.
- **Acao:** Adicionar unique constraint.

### 3. INTEGRACAO

**🔴 I-01: Nenhuma integracao de email implementada**
- **Problema:** Nao existe Edge Function para envio de email de convite. Brevo ja e usado em `send-task-reminders` mas nao foi replicado para convites.
- **Acao:** Criar `supabase/functions/send-invite-email/index.ts` usando padrao existente do Brevo.

### 4. UX/VISUAL

**🟡 U-01: Sem empty state na lista de membros**
- **Arquivo:** `src/pages/sellers.tsx:33-35`
- **Problema:** Se nao ha membros, mostra espaco em branco.
- **Acao:** Adicionar empty state: "Nenhum membro na equipe. Convide alguem!"

**🟡 U-02: Sem indicacao de expiracao do link**
- **Arquivo:** `src/components/sellers/invite-member-modal.tsx`
- **Problema:** Link expira em 7 dias mas UI nao informa isso.
- **Acao:** Mostrar "Link valido por 7 dias" ao exibir o link.

**🟡 U-03: Sem opcao de copiar/reenviar link da lista de pendentes**
- **Arquivo:** `src/pages/sellers.tsx:41-62`
- **Problema:** Admin ve convites pendentes mas nao pode copiar o link novamente. Precisa criar novo convite.
- **Acao:** Adicionar botao de copiar link ao lado de cada convite pendente.

**🟢 U-04: Pagina de aceitacao sem proximos passos**
- **Arquivo:** `src/pages/aceitar-convite.tsx`
- **Problema:** Apos aceitar, mostra "Convite aceito! Redirecionando..." sem orientacao.
- **Acao:** Adicionar mensagem com proximos passos antes do redirect.

### 5. COMERCIAL

**🟠 C-01: Fluxo de convite nao e autonomo**
- **Problema:** Admin nao consegue convidar e esquecer. Precisa copiar link e enviar manualmente. Para um CRM que vende produtividade, isso e anti-padrao.
- **Acao:** Email automatico resolve. E o gap mais importante para demonstracao.

**🟡 C-02: Sem notificacao ao admin quando convite e aceito**
- **Problema:** Admin nao sabe quando novo membro entrou. Precisa checar manualmente.
- **Acao:** Notificacao in-app via tabela `notifications` existente.
