## Dimensao 1: Funcional

### 1.1 CRITICO — DnD dropping em cards filhos, nao na coluna
**Severidade:** CRITICO
**Arquivo:** `src/pages/tarefas.tsx`
**Problema:** O `handleDragEnd` usa `over.id` e espera que seja um TaskStatus ('pending'/'in_progress'/'done'). Porem ao dropar sobre um card existente, `over.id` sera o UUID do task, nao o status da coluna. O drop so funciona quando se solta no espaco vazio da coluna (que tem o droppable id = status), nao sobre outros cards.
**Correcao:** No `handleDragEnd`, se `over.id` nao for um status valido, buscar o task com aquele id e usar seu status como target. Alternativamente, verificar `over.data.current` para pegar o container.

### 1.2 ALTO — Drag conflita com click no card
**Severidade:** ALTO
**Arquivo:** `src/components/tarefas/task-card.tsx`
**Problema:** O card tem `onClick={() => onEdit(task)}` junto com drag listeners/attributes no mesmo elemento. Com `activationConstraint: { distance: 8 }`, cliques rapidos abrem o modal de edicao, mas cliques lentos (com micro-movimento de mouse) podem iniciar drag e ao soltar abrir o modal tambem.
**Correcao:** Separar o area de drag do click, ou usar `isDragging` para suprimir o click.

### 1.3 ALTO — Coluna "Feito" acumula infinitamente
**Severidade:** ALTO
**Arquivo:** `src/services/tasks.service.ts`
**Problema:** A query filtra `.neq('status', 'cancelled')` mas nao limita tarefas `done`. Com o tempo, a coluna "Feito" vai acumular centenas de cards. A query retorna tudo.
**Correcao:** Limitar tarefas `done` aos ultimos 30 dias (`completed_at > now - 30 days`) ou adicionar paginacao/limite.

### 1.4 MEDIO — CreateTaskModal nao reseta defaultValues ao reabrir
**Severidade:** MEDIO
**Arquivo:** `src/components/tarefas/create-task-modal.tsx`
**Problema:** `defaultValues` em `useForm` so e aplicado na montagem. Se o modal e fechado e reaberto (sem desmontar), o form mantem os valores da ultima abertura. O `reset()` e chamado no `onClose`, mas nao injeta os defaultValues novamente.
**Correcao:** Usar `useEffect` no `open` para chamar `reset({ type: 'todo', lead_id: defaultLeadId ?? '', assigned_to: profile?.id ?? '' })`.

### 1.5 MEDIO — useLeads() no create-task-modal puxa dados pesados
**Severidade:** MEDIO
**Arquivo:** `src/components/tarefas/create-task-modal.tsx`
**Problema:** `useLeads()` depende de `usePipelineStore` (filtros do pipeline) e `useTeamMembers()`. Muda os dados conforme filtros do pipeline mudam. Num contexto de tarefas isso e inesperado — o select de leads deve mostrar TODOS os leads, nao os filtrados pelo pipeline.
**Correcao:** Criar uma query leve `useAllLeadsSimple()` que busca apenas `id, name, phone` sem depender do pipeline store.

### 1.6 BAIXO — Dropdown no task-card abre junto com drag
**Severidade:** BAIXO
**Arquivo:** `src/components/tarefas/task-card.tsx`
**Problema:** O DropdownMenuTrigger tem `onClick stopPropagation` mas o drag listener esta no elemento pai. Em mobile/touch, o menu pode ser dificil de acionar sem iniciar drag.
**Correcao:** Considerar mover drag handle para um icone especifico em vez de todo o card.

## Dimensao 4: UX/Visual

### 4.1 ALTO — Nao ha estado de erro visivel
**Severidade:** ALTO
**Arquivo:** `src/pages/tarefas.tsx`
**Problema:** A pagina trata `isLoading` e empty state, mas nao trata `isError`. Se a query falha, a pagina fica vazia sem feedback.
**Correcao:** Adicionar tratamento de `isError` com mensagem e botao de retry, como no dashboard.

### 4.2 MEDIO — Cards na coluna "Feito" parecem identicos aos pendentes
**Severidade:** MEDIO
**Arquivo:** `src/components/tarefas/task-card.tsx`
**Problema:** Tarefas concluidas tem a mesma aparencia visual das pendentes. Deveria ter opacidade reduzida ou titulo riscado.
**Correcao:** Adicionar classe condicional para `task.status === 'done'`: opacity-60 e line-through no titulo.

### 4.3 MEDIO — Sem feedback visual durante drag entre colunas
**Severidade:** MEDIO
**Arquivo:** `src/pages/tarefas.tsx`
**Problema:** O pipeline usa `DragOverlay` para mostrar o card "fantasma" durante drag. O kanban de tarefas nao tem DragOverlay, entao o card some durante o arraste.
**Correcao:** Adicionar DragOverlay com copia visual do card arrastado.

### 4.4 BAIXO — Filtro de busca nao tem debounce
**Severidade:** BAIXO
**Arquivo:** `src/pages/tarefas.tsx`
**Problema:** A busca filtra client-side no `useMemo`, entao nao ha chamada API por keystroke. Mas em listas grandes, re-renders frequentes podem causar lag.
**Correcao:** Opcional — adicionar debounce de 300ms se necessario.
