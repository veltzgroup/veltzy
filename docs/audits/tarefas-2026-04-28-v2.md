### 1.1 BAIXO — Dropdown vs drag em mobile
**Status:** Aberto (backlog)
**Arquivo:** `src/components/tarefas/task-card.tsx`
**Nota:** Em touch, o drag listener no card inteiro pode interferir com o dropdown.
Mitigacao futura: mover drag handle para icone especifico.

### 1.2 BAIXO — Edge Function sdr-ai precisa suportar mode meeting-reminders
**Arquivo:** `src/components/tarefas/create-task-modal.tsx:159`
**Problema:** O modal chama `supabase.functions.invoke('sdr-ai', { body: { mode: 'meeting-reminders', ... } })` mas a Edge Function sdr-ai pode nao ter esse mode implementado. O fallback generico funciona, mas as mensagens nao sao personalizadas pela IA.
**Mitigacao:** O fallback ja gera mensagens aceitaveis. Implementar o mode no sdr-ai quando priorizado.

### 3.1 BAIXO — Edge Function create-calendar-event nao existe
**Arquivo:** `src/services/tasks.service.ts:145`
**Problema:** Se a empresa tem oauth_integrations com google_calendar, o service tenta chamar `create-calendar-event` que nao existe. O try/catch ignora o erro, mas logs no Supabase vao mostrar 404.
**Mitigacao:** try/catch ja trata. Criar a Edge Function quando Hub v0 estiver pronto.

## Dimensao 5: Comercial

### O modulo e demonstravel?
SIM. Fluxo completo funcional:
1. Criar tarefa vinculada a lead com tipo e prazo
2. Arrastar entre colunas (A fazer -> Em andamento -> Feito)
3. Ver banner de reuniao iminente
4. Receber lembretes automaticos (WhatsApp + email)
5. Copiloto IA notificando tarefas vencidas e leads sem contato
6. Badge de vencidas no menu alertando o vendedor
7. Hierarquia de visibilidade por role

### Diferencial competitivo
- Lembretes IA personalizados para reunioes
- Copiloto diario que detecta 4 cenarios criticos
- Integracao profunda com pipeline (aba no lead, badge no card)
- Kanban visual com DnD funcional
