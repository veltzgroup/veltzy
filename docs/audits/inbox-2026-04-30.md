## Achados por Dimensao

### 1. FUNCIONAL

#### 🔴 F1 — Migration 020 referencia enum inexistente
**Arquivo:** `supabase/migrations/020_internal_notes.sql`
**Problema:** Usa `ALTER TYPE veltzy.sender_type_enum` mas o enum se chama `veltzy.sender_type`. A migration falhou silenciosamente — o valor `'internal'` nunca foi adicionado ao enum.
**Impacto:** Nenhum impacto direto porque o codigo usa `sender_type = 'human'` + `is_internal = true` para notas internas (nao usa sender_type='internal'). Mas o TypeScript type `SenderType` inclui `'internal'` sem correspondente no banco.
**Acao:** Criar migration corretiva renomeando para `ALTER TYPE veltzy.sender_type ADD VALUE IF NOT EXISTS 'internal'`. Ou remover `'internal'` do TypeScript type ja que nao e usado.

#### 🟡 F2 — Notas internas aparecem no preview da conversa
**Arquivo:** `supabase/migrations/021_sla_fields.sql` (RPC get_conversation_list, linhas 96-102)
**Problema:** A lateral join que busca a ultima mensagem nao filtra `is_internal = false`. Se a ultima mensagem de uma conversa for uma nota interna, ela aparece no preview da lista.
**Impacto:** Vendedor ve "conteudo da nota" como preview em vez da ultima mensagem real do lead.
**Acao:** Adicionar `AND (m.is_internal IS NOT TRUE)` na lateral join da RPC.

#### 🟡 F3 — first_response_at nunca e populado
**Arquivo:** `supabase/migrations/021_sla_fields.sql`, `src/services/messages.service.ts`
**Problema:** O campo `first_response_at` foi criado na migration mas nenhum codigo o popula. Nem o webhook, nem o service, nem trigger.
**Impacto:** Metrica de tempo de primeira resposta indisponivel.
**Acao:** Atualizar `sendMessage` e `routeMessage` para setar `first_response_at = NOW()` quando `first_response_at IS NULL` e sender_type = 'human'.

#### 🟡 F4 — Sem virtualizacao na lista de mensagens
**Arquivo:** `src/components/inbox/message-list.tsx`
**Problema:** Renderiza todas as mensagens de uma vez. Conversas com 200+ mensagens vao causar lag.
**Impacto:** Performance degradada em conversas longas.
**Acao:** Implementar react-virtual ou react-window. Prioridade media — nao bloqueia demo.

### 2. DADOS

#### 🔴 D1 — SLA hardcoded em 30 minutos
**Arquivo:** `supabase/functions/check-sla/index.ts` (linha 18)
**Problema:** `const SLA_MINUTES = 30` hardcoded. Deveria vir de `system_settings` > `business_rules` > `sla_hours` que ja existe no admin.
**Acao:** Buscar `sla_hours` da tabela `system_settings` e converter para minutos.

#### 🟡 D2 — Realtime so escuta INSERT, nao UPDATE
**Arquivo:** `src/hooks/use-messages.ts` (linhas 26-41)
**Problema:** Subscription so reage a `event: 'INSERT'`. Se uma mensagem for atualizada (transcricao de audio, edicao futura), o cliente nao reflete a mudanca.
**Impacto:** Transcricoes de audio via Whisper nao aparecem em tempo real — so no proximo refetch.
**Acao:** Adicionar listener para `event: 'UPDATE'` tambem.

#### 🟡 D3 — Polling de 15s na lista de conversas
**Arquivo:** `src/hooks/use-conversation-list.ts` (linha 17)
**Problema:** `refetchInterval: 15000` — poll a cada 15s em vez de usar Realtime.
**Impacto:** Nova mensagem pode demorar ate 15s para aparecer na lista lateral.
**Acao:** Toleravel para MVP. Migrar para Realtime subscription em fase futura.

#### 🟡 D4 — Signed URL com TTL de 1 hora para arquivos
**Arquivo:** `src/components/inbox/chat-input.tsx` (linha 69)
**Problema:** `createSignedUrl(path, 3600)` — URL expira em 1h. Se a mensagem ficar na fila (message_queue) por mais de 1h, o link quebra.
**Impacto:** Arquivos enviados via fila de automacao podem falhar.
**Acao:** Aumentar TTL para 24h ou gerar URL no momento do envio (process-message-queue).

### 3. INTEGRACOES

#### 🟡 I1 — zapi-send nao valida resposta da Z-API
**Arquivo:** `supabase/functions/zapi-send/index.ts` (~linha 103)
**Problema:** Faz `fetch` para Z-API mas nao verifica `res.ok`. Se Z-API retorna erro, a mensagem e salva no banco como se tivesse sido enviada.
**Acao:** Verificar `res.ok` e retornar erro ao cliente se falhar.

#### 🟡 I2 — Avatar fetch assume numero brasileiro
**Arquivo:** `supabase/functions/zapi-webhook/index.ts` (linha 175)
**Problema:** `const intlPhone = phone.length === 11 ? '55${phone}' : phone` — assume que numero com 11 digitos e brasileiro.
**Impacto:** Nulo para o Veltzy hoje (100% Brasil). Relevante se expandir internacionalmente.
**Acao:** Baixa prioridade. Documentar limitacao.

#### 🟢 I3 — Avatar fetch sem timeout
**Arquivo:** `supabase/functions/zapi-webhook/index.ts` (linha 177)
**Problema:** Fetch do avatar nao tem timeout. Se Z-API travar, o bloco try/catch garante que nao bloqueia o webhook (esta dentro de try separado), mas a Edge Function pode consumir tempo.
**Acao:** Adicionar `AbortController` com timeout de 5s.
